# Framework Installation Guide

## Quick Install

```bash
curl -fsSL https://raw.githubusercontent.com/silvanoneto//main/install.sh | bash
```

Or clone and install locally:

```bash
git clone https://github.com/silvanoneto/.git
cd 
chmod +x install.sh
./install.sh
```

## Prerequisites

- **Rust** 1.70+ (with Cargo)
- **Git**
- **Node.js** 18+ (optional, for VSCode extension)

### Install Rust

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
```

## What Gets Installed

The installation script will:

1. ✅ Clone the repository to `~/.`
2. ✅ Build all components (sil-core, lis-core, lis-cli, lis-format)
3. ✅ Install binaries to `~/.cargo/bin/`:
   - `lis` - LIS compiler and runner
   - `lis-format` - Code formatter
4. ✅ Create configuration at `~/.config//config.toml`

## Manual Installation

### 1. Clone Repository

```bash
git clone https://github.com/silvanoneto/.git
cd 
```

### 2. Build Framework

```bash
# Build everything
cargo build --release

# Or build specific components
cargo build --release -p sil-core
cargo build --release -p lis-core
cargo build --release -p lis-cli
cargo build --release -p lis-format
```

### 3. Install Binaries

```bash
# Install LIS CLI
cargo install --path lis-cli

# Install LIS Formatter
cargo install --path lis-format
```

### 4. Verify Installation

```bash
lis --version
lis-format --version
```

## Post-Installation

### Add to PATH

Ensure `~/.cargo/bin` is in your PATH:

```bash
# Add to ~/.bashrc or ~/.zshrc
export PATH="$HOME/.cargo/bin:$PATH"

# Reload shell
source ~/.bashrc  # or source ~/.zshrc
```

### VSCode Extension (Optional)

```bash
cd sil-vscode
npm install
npm run compile

# Install extension
code --install-extension .

# Or package as VSIX
npm run package
code --install-extension sil-language-*.vsix
```

## Quick Start

### 1. Create Your First LIS Program

```bash
cat > hello.lis <<'EOF'
fn main() {
    let x = 42;
    return x;
}
EOF
```

### 2. Run It

```bash
lis run hello.lis
```

Output:
```
   Compiling hello.lis
  Assembling Generated assembly (XX bytes)
     Running Generated bytecode (XX bytes)
    Finished Execution completed

Final State:
  L0 (Photonic):     ByteSil(ρ=X, θ=Y)
  ...
```

### 3. Format Code

```bash
# Format to stdout
lis-format hello.lis

# Format in-place
lis-format -w hello.lis

# Check if formatted
lis-format --check hello.lis
```

### 4. Compile Only

```bash
# Compile to SIL assembly
lis compile hello.lis

# Print assembly
lis compile hello.lis -p

# Check syntax
lis check hello.lis
```

## Configuration

Edit `~/.config//config.toml`:

```toml
[sil]
mode = "SIL-128"
enable_gpu = true
enable_npu = true

[lis]
optimization = "O2"
target_mode = "SIL-128"

[format]
indent_style = "spaces"
indent_size = 4
max_width = 100
```

## Updating

```bash
# Update to latest version
cd ~/.
git pull
cargo build --release
cargo install --path lis-cli --force
cargo install --path lis-format --force
```

Or use the update script:

```bash
~/./update.sh
```

## Uninstalling

```bash
# Remove installation
rm -rf ~/.

# Uninstall binaries
cargo uninstall lis lis-format

# Remove config
rm -rf ~/.config/

# Remove VSCode extension
code --uninstall-extension sil-.sil-language
```

## Troubleshooting

### Command not found

Make sure `~/.cargo/bin` is in your PATH:

```bash
echo $PATH | grep -q ".cargo/bin" || echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### Build errors

Update Rust to the latest stable:

```bash
rustup update stable
```

### GPU/NPU not working

Check your hardware support:

```bash
# Disable GPU/NPU in config if not available
lis info  # Shows available backends
```

Edit `~/.config//config.toml`:
```toml
[sil]
enable_gpu = false
enable_npu = false
```

### VSCode extension issues

```bash
cd ~/./sil-vscode
npm install
npm run compile
code --install-extension . --force
```

## Architecture

```
LIS Source (.lis)
     ↓ lis compile
SIL Assembly (.sil)
     ↓ assembler
Bytecode (.silc)
     ↓ VSP runtime
Execution
```

## Components

- **sil-core** - Core SIL types and VSP runtime
- **lis-core** - LIS language compiler
- **lis-cli** - Command-line interface
- **lis-format** - Code formatter
- **sil-vscode** - VSCode extension
